# Contribuir al repositorio

Gracias por querer contribuir a este repositorio. Aquí unas reglas y pasos recomendados.

1) Flujo de trabajo
- Usa ramas con prefijos: `feature/`, `fix/`, `chore/`.
- Abre Pull Requests hacia `main` para revisiones.
- Incluye un título descriptivo y una breve descripción en español.

2) Reglas de commits
- Preferir mensajes claros. Ejemplo: `feat: añadir dashboard3` o `fix: corregir typo en index.html`.

3) Archivos grandes
- No subir archivos grandes sin coordinarlos (usa Git LFS para .xcf, .psd, .mp4, etc.).

4) Revisiones
- Asigna revisores y añade capturas/screenshots cuando correspondan.

5) Tests y Calidad
- Si añades código automatizado, incluye pruebas básicas y asegura que la acción de CI pasa.

6) Contacto
- Para dudas, abre un issue y etiqueta `help wanted`.
